Managing unbalanced data for building machine learning models

Posted by Anne Dwyer on Tue, Aug 11, 2015 @ 08:00 AM

  • Tweet

 

What is a two-class data science problem?

In the world of data science, the two label classification (more simply, the two-class) problem is very common. The typical well-known example is the spam/not spam email problem. There are many other examples. In the airline industry, on time/delayed data is collected. In medicine, a test for a disease shows positive or negative.

The two-class problem has a binary response variable. Classification machine learning algorithms are used to learn the data and predict the class of new data. Data is split into training and test sets. The learning is done on the training set and the checked on the unseen test set. There are established tools for constructing the training and test sets.

How to handle unbalanced 2-class problems

But datasets seldom come with evenly distributed samples from either classes. For example, 90% of the available training data may have only one class of response variable (the majority class). Such datasets are called unbalanced data, which in the real world are quite common. An example is server process failure rate. The overall failure rate of a process on a set of servers may be very low but each failure is significant and triggers emergency action. A business case can be made for predicting when a process will fail. Analyzing the unbalanced dataset yields suboptimal results since the algorithm will predict no failure for every example. The Type 1 error is unacceptably high. Type 1 error occurs when a failure occurs but the model predicts no failure (also known as a false negative).

There are several options for learning unbalanced data. Many involve special sampling techniques to increase the signal of the minority class. One of the most common ways is to create a balanced training set with equal numbers of minority and majority class examples. There are two ways of doing this: downsampling and upsampling.

  • Downsampling creates a balanced dataset by matching the number of samples in the minority class with a random sample from the majority class.
  • Upsampling matches the number of samples in the majority class with resampling from the minority class.
But while it makes sense to learn the data with a balanced training set, it doesn’t make any sense to check the real world performance of the model on a balanced test set. The answer is to create a balanced training set and an unbalanced, unseen test set. This is not a trivial problem. Most open source statistical software for machine learning create the training/test split using stratified sampling which will return an unbalanced training set to match the unbalanced test set. The statistical software program R will allow you to downsample or upsample the data, but splitting this will give you a balanced training set and a balanced test set.

Here is a simple illustration. Suppose you have a dataset that measures some type of performance.  The dataset has 10000 examples and shows a 5% failure rate. There are 500 failure (minority class) examples and 9500 success (majority class) examples.  Suppose that you want to use 80% of the examples for training and 20% for testing. A stratified train/test split would give you a training set with 8000 examples consisting of  400 examples from the minority class and 7600 examples from the majority class. The test set would have 2000 examples with 100 from the minority class and 1900 from the majority class. Splitting a downsample dataset would give a balanced training set of 800 examples with 400 minority class examples and 400 majority class examples and a balanced test set with 200 examples of 100 samples from each class. What we really needed is a balanced training set with 800 total samples and an unbalanced unseen test set with 2000 examples split into 100 minority class examples and 1900 majority class examples. Preparing datasets with such distributions also comes in handy during model optimization phases.

Implementing the balanced training-unbalanced/unseen testing in practice

I have written Python code which automates the process. The program takes in a dataset and returns a balanced training set and an unbalanced, unseen test set. The procedure samples the majority and minority classes separately. A random order index is generated for the minority class. The first part of this index is used to create half of the balanced training set and the rest is used for the unbalanced test set.  The process for the majority class dataset is more complicated since the total index size must be calculated for the unbalanced set and added to the index size for the balanced set. Once the index is generated, it is used to split the majority class. The final step merges and shuffles the datasets. This code came in very handy for many predictive maintenance type of applications.

The datasets are numpy arrays. The arrays can be used as inputs to the scikit learn machine learning algorithms. They can also be converted to pandas data frames and output as csv files.
